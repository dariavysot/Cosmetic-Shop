# Завдання до проєкту: "Магазин косметики"

**Дисципліна:** SQL  
**Тип проєкту:** Web-застосунок  

## Опис
Створити веб-застосунок «Магазин косметики», який працюватиме з базою даних та включатиме такі основні таблиці:

### Таблиці

#### 1. Косметика
| Поле        | Тип даних | Опис           |
|------------|-----------|----------------|
| Id         | INT       | Унікальний ідентифікатор косметики |
| Назва      | VARCHAR   | Назва косметичного продукту |
| Постачальник | VARCHAR | Назва постачальника |
| Ціна       | DECIMAL   | Ціна продукту |

#### 2. Постачальник
| Поле        | Тип даних | Опис |
|------------|-----------|------|
| Id         | INT       | Унікальний ідентифікатор постачальника |
| Постачальник | VARCHAR | Назва постачальника |

#### 3. Склад магазину
| Поле        | Тип даних | Опис |
|------------|-----------|------|
| Id         | INT       | Унікальний ідентифікатор запису |
| Косметика  | VARCHAR   | Назва косметики |
| Кількість  | INT       | Поточна кількість на складі |

> При потребі можна додавати додаткові таблиці.

---

## Функціональні вимоги
В застосунку повинно бути передбачено наступні можливості:

1. Перегляд залишків на складі.
2. Формування купівельної корзини покупця:
   - що за косметику купив;
   - ціна;
   - сума покупки.
3. Редагування купівельної корзини.
4. Автоматичне списання кількості косметики при купівлі.
5. Додавання косметики на склад.
6. Пошук косметики за постачальником.
7. Пошук косметики за назвою.
8. Дружній інтерфейс.

---

## Примітки
- Проєкт реалізується як веб-застосунок.
- Можна додавати власні таблиці та поля для розширення функціональності.
